# دليل النشر — الاستقدام الذكي (smartistiqdam.com)

نظام **Node.js (Express) + قاعدة بيانات MySQL**. يُخدّم منه كل شيء: صفحة الهبوط، لوحة المكتب، لوحة مدير المنصة، والـ API. قاعدة البيانات خارجية (MySQL) لضمان بقاء البيانات عبر عمليات إعادة النشر.

> **GitHub** = مستودع الكود (ومصدر النشر). **هوستنجر (خطة Business)** = تشغيل النظام + قاعدة MySQL.

## المتطلبات
- خطة هوستنجر تدعم تطبيقات Node.js (Business فأعلى).
- Node.js 18 أو أحدث (هوستنجر يوفّر 18/20/22/24).
- قاعدة بيانات MySQL (مضمّنة في خطة Business).

---

## 1) إنشاء قاعدة بيانات MySQL في هوستنجر
من hPanel → **Databases → MySQL Databases**:
1. أنشئ قاعدة بيانات جديدة، ومستخدماً لها بكلمة مرور قوية.
2. اربط المستخدم بالقاعدة وامنحه كل الصلاحيات.
3. سجّل القيم: اسم القاعدة، اسم المستخدم، كلمة المرور، والمضيف (غالباً `localhost`).

## 2) الرفع على GitHub
```bash
cd مجلد-المشروع
git init
git add .
git commit -m "الاستقدام الذكي - MySQL"
git branch -M main
git remote add origin https://github.com/<حسابك>/smartistiqdam.git
git push -u origin main
```
ملف `.gitignore` يتجاهل `node_modules` و`.env` والمرفقات والسجلات.

## 3) النشر على هوستنجر (Node.js)
من hPanel → **Websites → Add Website → Node.js Apps**:
1. اختر **Import Git Repository** (أو رفع ملف ZIP).
2. إصدار Node: 18+، وأمر التشغيل: `npm start` (ملف البداية `src/server.js`).
3. **متغيّرات البيئة (Environment Variables):** أضِف:
   - `APP_URL=https://smartistiqdam.com`
   - `JWT_SECRET=` قيمة عشوائية طويلة.
   - `SUPERADMIN_USER` و`SUPERADMIN_PASS` (كلمة مرور قوية).
   - `DB_HOST` (غالباً `localhost`)، `DB_PORT=3306`، `DB_NAME`، `DB_USER`، `DB_PASSWORD` (من الخطوة 1).
   - `UPLOADS_DIR=./data/uploads`
   - `SEED_DEMO=false` (لمنع المكاتب التجريبية).
4. **Deploy** — سيُنشئ النظام كل الجداول ويزرع حساب المدير تلقائياً عند أول تشغيل.

## 4) الدومين وHTTPS
- اربط `smartistiqdam.com` بالتطبيق من إعدادات الموقع.
- فعّل شهادة SSL المجانية (Let's Encrypt) من hPanel، ثم التحويل الإجباري إلى https.

## 5) ضبط النظام بعد النشر
افتح `https://smartistiqdam.com/admin.html` وادخل بحساب المدير، ثم **الإعدادات**:
1. **بيانات الشركة:** الاسم/الإيميل/الواتساب/الجوال/السجل التجاري/الرقم الضريبي — تظهر على صفحة الهبوط وفي الفواتير.
2. **بوابة الدفع:** أدخل **Tap Secret Key** لتفعيل الدفع وروابط السداد.
3. **إعدادات الاشتراك:** مدة التجربة وأسعار الخطط.
4. **المقالات:** انشر مقالات تظهر في مدونة صفحة الهبوط.
5. أنشئ المكاتب وحساباتها من تبويب **المكاتب**.

## ملاحظات مهمة
- **النسخ الاحتياطي:** قاعدة MySQL تحوي كل بياناتك — فعّل نسخاً احتياطياً دورياً من هوستنجر.
- **المرفقات (صور/سير/مستندات):** تُحفظ في `UPLOADS_DIR`. اجعله مساراً دائماً، وانتبه أن إعادة بناء التطبيق قد لا تُبقي ملفات المجلد إن كان داخل مجلد النشر؛ يُفضّل لاحقاً نقلها إلى تخزين خارجي إن زاد حجمها.
- غيّر كلمة مرور مدير المنصة قبل أول تشغيل عبر `SUPERADMIN_PASS`.

## (اختياري) صفحة هبوط ثابتة على GitHub Pages
يمكن رفع `public/landing.html` باسم `index.html` مع `public/assets/` وملف `CNAME` للنطاق، لكن المزايا الديناميكية (المقالات/الشركة/الدفع/الدخول) تحتاج الخادم على هوستنجر.
